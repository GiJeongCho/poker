# ♠️ 8인 텍사스 홀덤 포커 (Texas Hold'em Poker)

최대 8명이 함께 즐길 수 있는 실시간 멀티플레이어 웹 기반 텍사스 홀덤 포커 게임입니다.
Python FastAPI와 Socket.IO를 사용하여 구현되었습니다.

---

## 📌 주요 기능

| 기능 | 설명 |
|---|---|
| **실시간 멀티플레이** | 최대 8명까지 한 방에서 동시 게임 참여 |
| **관전 모드** | 게임 진행 중 입장 시 자동 관전, 다음 라운드부터 참여 |
| **자동 진행** | 프리플롭 → 플롭 → 턴 → 리버 → 쇼다운 순서로 단계 진행 |
| **족보 확인** | 게임 중 언제든지 포커 족보(Ranking) 팝업 확인 가능 |
| **반응형 시점** | 자신의 카드가 항상 테이블 하단 중앙에 위치하도록 자동 정렬 |
| **개인화된 패 공개** | 자신의 패만 앞면으로 표시, 상대 패는 뒤집힌 채로 표시 |
| **쇼다운 공개** | 쇼다운 단계에서 폴드하지 않은 모든 플레이어의 패 자동 공개 |
| **동점 처리** | 동점 시 팟을 균등 분배 |
| **방장 자동 위임** | 방장이 나갈 경우 다음 플레이어에게 방장 권한 자동 이전 |

---

- **Frontend**: HTML5, CSS3, Vanilla JavaScript (Socket.IO Client v4.5.1)

## 🚀 설치 및 실행 방법

이 프로젝트는 `uv`로 가상환경 생성과 패키지 설치를 모두 처리합니다.

### 0. uv 설치 (최초 1회)

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 1. 환경 설정

```bash
# 1. 프로젝트 폴더로 이동
cd /path/to/poker

# 2. 가상환경 생성 (Python 3.13)
uv venv --python 3.13

# 3. 가상환경 활성화
source .venv/bin/activate

# 4. 의존성 패키지 설치
uv pip install -r backend/requirements.txt
```

### 2. 서버 실행

**방법 A (권장): Python 스크립트로 실행**
```bash
source .venv/bin/activate
python backend/main.py
```

**방법 B: Uvicorn 직접 실행 (개발 모드 - 코드 변경 자동 반영)**
```bash
source .venv/bin/activate
uvicorn backend.main:socket_app --host 0.0.0.0 --port 8092 --reload
```

### 3. 게임 접속

브라우저를 열고 아래 주소로 접속합니다.

```
http://localhost:8092
```

같은 네트워크의 다른 기기에서 접속할 경우:
```
http://<서버 IP>:8092
```

## 🎮 게임 진행 방법

### 1단계: 입장

1. 닉네임 입력란에 이름을 입력합니다.
2. **방 만들기**: 새 방을 생성하고 방장으로 입장합니다. 화면 상단에 **방 코드** (`room_XXXX` 형식)가 표시됩니다.
3. **참가하기**: 방 코드를 입력하고 기존 방에 참가합니다.

> **방 코드는 친구들에게 공유하여 함께 참여할 수 있습니다.**

### 2단계: 게임 시작

- 최소 **2명** 이상 입장 시 방장이 **[게임 시작]** 버튼을 눌러 게임을 시작합니다.
- 각 플레이어는 게임 시작 시 **1,000 칩**을 보유합니다.
- 칩은 라운드가 끝나도 유지됩니다(리셋되지 않음).

### 3단계: 게임 라운드 진행

현재 버전은 **방장이 수동으로 단계를 진행**하는 방식입니다.
방장 화면에서 **[다음 단계 (관리자)]** 버튼을 눌러 각 단계를 넘깁니다.

| 단계 | 설명 | 공개 카드 수 |
|---|---|---|
| **프리플롭 (Pre-Flop)** | 각 플레이어에게 홀 카드 2장 배분 | 0장 |
| **플롭 (Flop)** | 커뮤니티 카드 3장 공개 | 3장 |
| **턴 (Turn)** | 커뮤니티 카드 1장 추가 공개 | 4장 (누적) |
| **리버 (River)** | 커뮤니티 카드 1장 추가 공개 | 5장 (누적) |
| **쇼다운 (Showdown)** | 모든 패 공개, 승자 결정 및 팟 분배 | - |

> 각 단계마다 1장의 카드를 버닝(Burn)한 후 커뮤니티 카드를 오픈합니다.

### 4단계: 승자 결정

- 쇼다운 단계에서 폴드하지 않은 플레이어의 **홀 카드 2장 + 커뮤니티 카드 5장(총 7장)** 중 가장 높은 5장 조합으로 족보가 결정됩니다.
- 최고 족보를 가진 플레이어가 팟 전체를 가져갑니다.
- **동점 시 팟을 균등 분배**합니다.
- 쇼다운 후 자동으로 대기 상태로 전환되며, 방장이 다시 시작 버튼을 눌러 다음 판을 시작합니다.

### 관전 모드

- 게임 진행 중 입장한 플레이어는 자동으로 **관전 모드**로 입장합니다.
- 해당 라운드 동안 카드를 받지 않으며, **다음 라운드부터 정식 참여**합니다.
- 관전 중인 플레이어의 자리에는 "관전 중" 텍스트가 표시됩니다.

## 🃏 포커 족보 (높은 순)

| 순위 | 족보 | 설명 | 예시 |
|---|---|---|---|
| 1 | **로얄 플러쉬** | 같은 무늬의 A, K, Q, J, 10 | ♠A ♠K ♠Q ♠J ♠10 |
| 2 | **스트레이트 플러쉬** | 같은 무늬의 연속된 5장 | ♥7 ♥8 ♥9 ♥10 ♥J |
| 3 | **포카드** | 같은 숫자의 카드 4장 | K♠ K♥ K♦ K♣ + any |
| 4 | **풀 하우스** | 트리플(3장) + 원페어(2장) | A A A + K K |
| 5 | **플러쉬** | 같은 무늬의 카드 5장 | ♦2 ♦5 ♦7 ♦9 ♦J |
| 6 | **스트레이트** | 연속된 숫자의 카드 5장 | 5 6 7 8 9 |
| 7 | **트리플** | 같은 숫자의 카드 3장 | Q Q Q + any |
| 8 | **투 페어** | 같은 숫자의 쌍이 2개 | J J + 8 8 + any |
| 9 | **원 페어** | 같은 숫자의 쌍이 1개 | A A + any |
| 10 | **하이 카드** | 위에 해당 없으면 가장 높은 카드 | A + K + 9 + 6 + 2 |

> **휠(Wheel) 스트레이트** (A-2-3-4-5)도 인정됩니다. 이 경우 5가 최고 카드입니다.

족보가 동일할 경우 **킥커(Kicker)** 카드로 순위를 결정합니다.

## 🔌 Socket.IO 이벤트 목록

### 클라이언트 → 서버

| 이벤트 | 데이터 | 설명 |
|---|---|---|
| `create_room` | `{}` | 새 방 생성 (응답으로 `room_id` 반환) |
| `join_room` | `{ room_id, name }` | 기존 방 참가 |
| `start_game` | `{}` | 게임 시작 (방장만 가능) |
| `next_phase` | `{}` | 다음 단계로 이동 (방장만 가능) |

### 서버 → 클라이언트

| 이벤트 | 설명 |
|---|---|
| `room_joined` | 방 참가 성공 시 `{ room_id, host_id }` 전달 |
| `game_state` | 게임 상태 전체 브로드캐스트 (개인화된 패 포함) |
| `error` | 에러 메시지 전달 (`{ message }`) |

### `game_state` 페이로드 구조

```json
{
  "room_id": "room_1234",
  "state": "FLOP",
  "pot": 0,
  "community_cards": [{ "suit": "♠", "rank": "A", "value": 14 }, ...],
  "logs": ["최근 로그 최대 5줄"],
  "host_id": "socket_id",
  "players": [
    {
      "id": "socket_id",
      "name": "닉네임",
      "chips": 1000,
      "is_folded": false,
      "current_bet": 0,
      "is_all_in": false,
      "hand": [{ "suit": "♥", "rank": "K", "value": 13 }, ...],
      "best_rank": "ONE_PAIR"
    }
  ]
}
```

> `hand` 필드: 본인에게는 실제 카드가 전송되고, 타인에게는 `{ "suit": "?", "rank": "?" }` 형식의 뒷면 데이터가 전송됩니다.

## ⚠️ 현재 버전의 제한 사항

이 프로젝트는 **간소화된 데모 버전**으로, 다음 기능들은 아직 구현되지 않았습니다.

- **배팅 액션 없음**: 콜(Call), 레이즈(Raise), 폴드(Fold) 버튼이 없습니다. 게임은 자동 흐름으로 진행됩니다.
- **블라인드(Blind) 없음**: 스몰 블라인드/빅 블라인드 시스템이 없으며, 팟은 0에서 시작합니다.
- **수동 단계 전환**: 방장이 직접 [다음 단계] 버튼을 눌러야 다음 단계로 넘어갑니다. 타이머 자동 진행은 없습니다.
- **재접속 불가**: 연결이 끊어진 플레이어는 해당 방에서 제거되며, 재접속 기능이 없습니다.
- **칩 없는 플레이어 처리**: 칩이 0인 플레이어는 자동으로 폴드 상태로 처리되나, 퇴장 처리되지 않습니다.
- **올인(All-in) 사이드팟**: 올인 플레이어에 대한 사이드 팟 계산이 없습니다.

## 📝 개발 노트

- 서버는 `0.0.0.0:8092` 포트에서 실행됩니다.
- 방 코드 형식: `room_XXXX` (1000~9999 범위의 랜덤 4자리 숫자)
- `game_state` 브로드캐스트 시 각 플레이어마다 개별 emit하여 패 정보를 개인화합니다.
- 게임 로그는 최근 **5줄**만 클라이언트에 전달됩니다.
- 프론트엔드는 별도 빌드 과정 없이 단일 `index.html`로 동작합니다.
