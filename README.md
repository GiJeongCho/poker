# ♠️ 8인 텍사스 홀덤 포커 (Texas Hold'em Poker)

최대 8명이 함께 즐길 수 있는 실시간 멀티플레이어 웹 기반 텍사스 홀덤 포커 게임입니다.
Python FastAPI와 Socket.IO를 사용하여 구현되었습니다.

---

## 📌 주요 기능

| 기능 | 설명 |
|---|---|
| **실시간 멀티플레이** | 최대 8명까지 한 방에서 동시 게임 참여 |
| **관전 모드** | 게임 진행 중 입장 시 자동 관전, 다음 라운드부터 참여 |
| **블라인드 배팅** | 스몰/빅 블라인드 자동 차감, 콜/레이즈/폴드/체크/올인 지원 |
| **자동 단계 진행** | 배팅 라운드 완료 시 프리플롭→플롭→턴→리버→쇼다운 자동 진행 |
| **채팅** | 게임 중 실시간 채팅 (오른쪽 패널) |
| **족보 확인** | 게임 중 언제든지 포커 족보 팝업 확인 가능 |
| **반응형 시점** | 자신의 카드가 항상 테이블 하단 중앙에 위치하도록 자동 정렬 |
| **개인화된 패 공개** | 자신의 패만 앞면으로 표시, 상대 패는 뒤집힌 채로 표시 |
| **쇼다운 공개** | 쇼다운 단계에서 폴드하지 않은 모든 플레이어의 패 자동 공개 |
| **동점 처리** | 동점 시 팟을 균등 분배 |
| **방장 자동 위임** | 방장이 나갈 경우 다음 플레이어에게 방장 권한 자동 이전 |

## 🛠️ 기술 스택

- **Backend**: Python 3.13, FastAPI, python-socketio, Uvicorn
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (Socket.IO Client v4.5.1)

## 🚀 설치 및 실행 방법

이 프로젝트는 `uv`로 가상환경 생성과 패키지 설치를 모두 처리합니다.

### 0. uv 설치 (최초 1회)

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 1. 환경 설정 (최초 1회)

```bash
# 1. 프로젝트 폴더로 이동
cd /path/to/poker

# 2. 가상환경 생성 (Python 3.13)
uv venv --python 3.13

# 3. 가상환경 활성화
source .venv/bin/activate

# 4. 의존성 패키지 설치
uv pip install -r backend/requirements.txt
```

### 2. 서버 실행

```bash
cd /path/to/poker
source .venv/bin/activate
python backend/main.py
```

개발 모드 (코드 변경 자동 반영):
```bash
source .venv/bin/activate
uvicorn backend.main:socket_app --host 0.0.0.0 --port 8094 --reload
```

### 3. 게임 접속

```
http://localhost:8094
```

같은 네트워크의 다른 기기에서 접속할 경우:
```
http://<서버 IP>:8094
```

## 🎮 게임 진행 방법

### 1단계: 입장

1. 닉네임을 입력합니다.
2. **방 만들기**: 새 방을 생성하고 방장으로 입장합니다. 화면 상단에 **방 코드** (`room_XXXX` 형식)가 표시됩니다.
3. **참가하기**: 방 코드를 입력하고 기존 방에 참가합니다.

> 방 코드를 친구들에게 공유하여 함께 참여할 수 있습니다.

### 2단계: 게임 시작

- 최소 **2명** 이상 입장 시 방장이 **[게임 시작]** 버튼을 눌러 게임을 시작합니다.
- 각 플레이어는 **1,000 칩**으로 시작합니다.
- 칩은 라운드가 끝나도 유지됩니다 (리셋 없음).
- 게임 시작 즉시 **스몰 블라인드(10칩) / 빅 블라인드(20칩)** 가 자동 차감됩니다.

### 3단계: 배팅 라운드

각 플레이어는 자기 턴이 돌아오면 아래 액션 중 하나를 선택합니다.
배팅이 모두 완료되면 **자동으로 다음 단계**로 넘어갑니다.

#### 🃏 배팅 액션 설명

| 버튼 | 조건 | 설명 |
|---|---|---|
| **폴드 (Fold)** | 언제나 가능 | 이번 판을 포기합니다. 베팅한 칩은 돌려받지 못하며, 쇼다운에 참여하지 않습니다. |
| **체크 (Check)** | 추가 베팅이 없을 때만 가능 | 베팅 없이 차례를 넘깁니다. 이미 누군가 레이즈했다면 체크할 수 없습니다. |
| **콜 (Call)** | 이전 베팅이 있을 때 | 현재 최고 베팅액에 맞춰 칩을 냅니다. 버튼에 필요 금액이 표시됩니다 (예: `콜 (20칩)`). |
| **레이즈 (Raise)** | 언제나 가능 | 입력창에 **추가 베팅액**을 입력하고 누릅니다. 최소 빅 블라인드(20칩) 이상 추가해야 합니다. 예: 현재 베팅 20칩 상태에서 `30` 입력 → 총 50칩으로 레이즈. 레이즈 후 다른 플레이어들은 다시 콜/폴드/재레이즈를 해야 합니다. |
| **올인 (All-in)** | 언제나 가능 | 보유한 칩 전부를 베팅합니다. |

> **내 턴일 때만** 버튼이 표시됩니다. 대기 중에는 `[플레이어명]님의 차례...` 메시지가 표시됩니다.

#### 게임 단계별 흐름

| 단계 | 설명 | 공개 커뮤니티 카드 |
|---|---|---|
| **프리플롭 (Pre-Flop)** | 홀 카드 2장 배분, 블라인드 포스트, UTG부터 배팅 | 0장 |
| **플롭 (Flop)** | 커뮤니티 카드 3장 공개, SB부터 배팅 | 3장 |
| **턴 (Turn)** | 커뮤니티 카드 1장 추가 공개, SB부터 배팅 | 4장 |
| **리버 (River)** | 커뮤니티 카드 1장 추가 공개, SB부터 배팅 | 5장 |
| **쇼다운 (Showdown)** | 모든 패 공개, 족보 비교 후 승자가 팟 획득 | — |

> 각 단계 시작 전 1장을 버닝(Burn)한 후 커뮤니티 카드를 오픈합니다.

### 4단계: 승자 결정

- 폴드하지 않은 플레이어의 **홀 카드 2장 + 커뮤니티 카드 5장 (총 7장)** 중 가장 높은 5장 조합으로 족보가 결정됩니다.
- 나머지가 모두 폴드하면 즉시 승리 (패를 공개하지 않아도 됩니다).
- **동점 시 팟을 균등 분배**합니다.
- 쇼다운 후 자동으로 대기 상태가 되며, 방장이 다시 [게임 시작]을 눌러 다음 판을 진행합니다.
- 매 판마다 딜러 위치가 한 칸씩 이동합니다.

### 관전 모드

- 게임 진행 중 입장하면 자동으로 **관전 모드**로 입장됩니다.
- 해당 라운드는 카드를 받지 않으며, **다음 라운드부터 정식 참여**합니다.
- 관전 중인 플레이어 자리에는 "관전 중" 텍스트가 표시됩니다.

### 💬 채팅

- 화면 오른쪽 채팅 패널에서 같은 방 플레이어들과 실시간 대화가 가능합니다.
- 메시지 입력 후 **Enter** 또는 **[전송]** 버튼을 누릅니다.
- 채팅은 게임 진행과 무관하게 언제든 사용할 수 있습니다.

## 🃏 포커 족보 (높은 순)

| 순위 | 족보 | 설명 | 예시 |
|---|---|---|---|
| 1 | **로얄 플러쉬** | 같은 무늬의 A, K, Q, J, 10 | ♠A ♠K ♠Q ♠J ♠10 |
| 2 | **스트레이트 플러쉬** | 같은 무늬의 연속된 5장 | ♥7 ♥8 ♥9 ♥10 ♥J |
| 3 | **포카드** | 같은 숫자의 카드 4장 | K♠ K♥ K♦ K♣ + any |
| 4 | **풀 하우스** | 트리플(3장) + 원페어(2장) | A A A + K K |
| 5 | **플러쉬** | 같은 무늬의 카드 5장 | ♦2 ♦5 ♦7 ♦9 ♦J |
| 6 | **스트레이트** | 연속된 숫자의 카드 5장 | 5 6 7 8 9 |
| 7 | **트리플** | 같은 숫자의 카드 3장 | Q Q Q + any |
| 8 | **투 페어** | 같은 숫자의 쌍이 2개 | J J + 8 8 + any |
| 9 | **원 페어** | 같은 숫자의 쌍이 1개 | A A + any |
| 10 | **하이 카드** | 위 족보 미해당 시 가장 높은 카드 | A + K + 9 + 6 + 2 |

> **휠(Wheel) 스트레이트** (A-2-3-4-5)도 인정됩니다. 이 경우 5가 최고 카드입니다.

족보가 동일할 경우 **킥커(Kicker)** 카드로 순위를 결정합니다.

## 🔌 Socket.IO 이벤트 목록

### 클라이언트 → 서버

| 이벤트 | 데이터 | 설명 |
|---|---|---|
| `create_room` | `{}` | 새 방 생성 (응답으로 `room_id` 반환) |
| `join_room` | `{ room_id, name }` | 기존 방 참가 |
| `start_game` | `{}` | 게임 시작 (방장만 가능) |
| `player_action` | `{ action, raise_to }` | 배팅 액션 (fold/check/call/raise/allin) |
| `send_message` | `{ message }` | 채팅 메시지 전송 |

### 서버 → 클라이언트

| 이벤트 | 설명 |
|---|---|
| `room_joined` | 방 참가 성공 시 `{ room_id, host_id }` 전달 |
| `game_state` | 게임 상태 전체 브로드캐스트 (개인화된 패 포함) |
| `chat_message` | 채팅 메시지 수신 `{ name, message }` |
| `error` | 에러 메시지 전달 `{ message }` |

### `game_state` 페이로드 구조

```json
{
  "room_id": "room_1234",
  "state": "FLOP",
  "pot": 30,
  "community_cards": [{ "suit": "♠", "rank": "A", "value": 14 }, ...],
  "logs": ["최근 로그 최대 5줄"],
  "host_id": "socket_id",
  "current_player_id": "socket_id",
  "current_bet": 20,
  "small_blind": 10,
  "big_blind": 20,
  "players": [
    {
      "id": "socket_id",
      "name": "닉네임",
      "chips": 980,
      "is_folded": false,
      "current_bet": 20,
      "is_all_in": false,
      "has_acted": true,
      "hand": [{ "suit": "♥", "rank": "K", "value": 13 }, ...],
      "best_rank": "ONE_PAIR"
    }
  ]
}
```

> `hand` 필드: 본인에게는 실제 카드, 타인에게는 `{ "suit": "?", "rank": "?" }` 뒷면 데이터가 전송됩니다.

## ⚠️ 현재 버전의 제한 사항

- **사이드팟 없음**: 올인 플레이어에 대한 사이드 팟 계산이 없습니다.
- **재접속 불가**: 연결이 끊어진 플레이어는 해당 방에서 제거되며, 재접속 기능이 없습니다.
- **칩 소진 시**: 칩이 0인 플레이어는 폴드 상태로 처리되나 자동 퇴장되지 않습니다.
- **타이머 없음**: 배팅 시간 제한이 없습니다.

## 📝 개발 노트

- 서버는 `0.0.0.0:8094` 포트에서 실행됩니다.
- 방 코드 형식: `room_XXXX` (1000~9999 범위의 랜덤 4자리 숫자)
- `game_state` 브로드캐스트 시 각 플레이어마다 개별 emit하여 패 정보를 개인화합니다.
- 게임 로그는 최근 **5줄**만 클라이언트에 전달됩니다.
- 프론트엔드는 별도 빌드 과정 없이 단일 `index.html`로 동작합니다.
